## 前提

**🔍 情報収集の優先順位**：
1. **inputフォルダの情報を最優先**：`input/自社/`、`input/競合/`、`input/市場/`の情報を統合活用
2. **ウェブリサーチで補完**：inputで不足する情報をウェブサイトから収集
3. **矛盾時はinput優先**：ウェブリサーチとinput情報が矛盾する場合は、input情報を採用

**📁 inputフォルダの統合活用**：
- `input/自社/`：自社の実際の顧客ジャーニーデータ、タッチポイント情報等を確認
- `input/競合/`：競合のカスタマージャーニー情報、タッチポイント戦略等を確認
- `input/市場/`：市場環境の変化が顧客行動に与える影響を確認
  - **重要**：将来の市場環境での顧客行動変化を考慮したジャーニー設計

**入力ファイル：**前段階で作成された以下のファイルを参照してください：
- 「output/[ルートドメイン名]/ターゲティング_[企業名].md」
- 「output/[ルートドメイン名]/競合調査_[企業名].md」
- 「output/[ルートドメイン名]/ポジショニング_[企業名].md」
- 「output/[ルートドメイン名]/ペルソナ_[企業名].md」
（※ルートドメイン名は自社URLから抽出。例：https://moltsinc.co.jp/ → moltsinc）

**🔍 統合カスタマージャーニー開発手法**：
1. **inputフォルダ情報の統合活用**：
   - `input/自社/`から実際の顧客ジャーニーデータ、タッチポイント情報を取得
   - `input/競合/`から競合のカスタマージャーニー情報、タッチポイント戦略を取得
   - `input/市場/`から市場変化が顧客行動に与える影響を分析
   - 情報源を明確に記録（どのファイルから取得したかを記載）

2. **ウェブリサーチによる補完分析**：
   前段階で作成されたファイルと、各社のウェブサイト直接調査を組み合わせて分析してください：
   - 自社サイトの詳細調査（トップページ、サービス、事例、料金、会社概要等）
   - 競合3社のサイト詳細調査（同様の主要ページを調査）
   - 公開情報に基づく事実ベースの客観的分析
   - 情報源（参照ページURL）の明確化

3. **市場変化を考慮したジャーニー設計**：
   - `input/市場/`の情報を基に、将来の市場環境での顧客行動変化を予測
   - 現在と将来のカスタマージャーニー比較を実施
   - 市場変化に対する顧客の適応力を分析

**出力先の設定：**
1. **自社URLからルートドメイン名を抽出**（既に作成されているフォルダと同じ）
2. **出力ファイル**：
   - 「output/[ルートドメイン名]/ジャーニー_企業 or ブランド名.md」
   - 「output/[ルートドメイン名]/ジャーニー_企業 or ブランド名.csv」（横列が購買行動プロセス、縦列が状態と刺激）

**重要：必ず横型マトリックス形式で作成すること**

## 依頼内容

与えられたペルソナ、ポジショニングに基づいて、包括的なカスタマージャーニーを構築してください。このジャーニーは、ターゲットユーザーの購買行動プロセスを明確に示し、効果的なマーケティング戦略の開発に役立つものとします。

また、前段階で作成されたペルソナの数だけ、それぞれのカスタマージャーニー（状況と刺激）を作成し、すべて同一態度プロセスで統一して、1つのファイルとしてジャーニーを作成してください。

### 構築の際の重要ポイント：

1. **日常と課題認識の分離**：
    - 日常生活/業務の状況と、課題認識を別々のステップとして明確に区分してください
    - 日常の中で潜在的なニーズがどのように存在するかを描写してください

2. **購買前と購買後の区分**：
    - ジャーニーを「購買前」と「購買後」に明確に分けてください
    - 各段階での異なる目標、行動、重視する価値の変化を反映させてください

3. **各段階での具体的行動**：
    - 各ステージでターゲットユーザーが取る具体的な行動を詳細に記述してください
    - ユーザーの意思決定プロセスに影響を与える要素を含めてください

4. **態度変容のための訴求軸**：
    - 各段階でユーザーの態度を変容させるための効果的な訴求軸を提案してください
    - ユーザーの各段階での関心事や懸念点に合わせた価値訴求ポイントを示してください

5. **多様なタッチポイント**：
    - 特定のツールやプラットフォームではなく、タッチポイントの「種類」や「カテゴリー」として幅広く提示してください
    - ユーザーが情報を得たり、ブランドと接触したりする様々な可能性を示してください

### ジャーニーの構成：

以下の構造に従って、段階ごとに情報を整理してください：

**横軸**：購買行動プロセス（各態度）※ペルソナファイルから抽出した共通の購買プロセスに基づいて設定
- ペルソナファイルの「共通の購買プロセス」セクションを参照し、実際の顧客行動に合わせた段階を設定
- 例：日常 → 課題発生 → 情報収集 → 検討 → お問合せ → 契約決定（業界・サービス特性に応じて5-8段階で調整）

**縦軸**：ペルソナ別の状況と刺激（訴求軸）

### CSVファイルの形式：

**重要：不要な空白列・行は一切含めず、コンパクトな形式で作成すること**

```
"企業名カスタマージャーニー",[段階1],[段階2],[段階3],[段階4],[段階5],[段階6〜8]

"ペルソナ1刺激",ペルソナ1向け訴求軸（15-20文字）,ペルソナ1向け訴求軸（15-20文字）,ペルソナ1向け訴求軸（15-20文字）,ペルソナ1向け訴求軸（15-20文字）,ペルソナ1向け訴求軸（15-20文字）,ペルソナ1向け訴求軸（15-20文字）
"ペルソナ2詳細",状況説明（190-200文字）,状況説明（190-200文字）,状況説明（190-200文字）,状況説明（190-200文字）,状況説明（190-200文字）,状況説明（190-200文字）
"ペルソナ2刺激",ペルソナ2向け訴求軸（15-20文字）,ペルソナ2向け訴求軸（15-20文字）,ペルソナ2向け訴求軸（15-20文字）,ペルソナ2向け訴求軸（15-20文字）,ペルソナ2向け訴求軸（15-20文字）,ペルソナ2向け訴求軸（15-20文字）
```

**注意**: [段階1]〜[段階6〜8]は、ペルソナファイルの「共通の購買プロセス」から抽出した実際の段階名を使用してください。

### 各要素の詳細：

- **状況**: ユーザーがその態度/段階で置かれている「状態」を190-200文字で詳細に記述
  - ペルソナの業界・役職・課題を反映した現実的な描写
  - 具体的な業務状況や心理状態を含める
  - **inputフォルダまたはウェブリサーチによる事例ページから実際の顧客状況を参考**
  - **市場変化の影響を考慮**：将来の市場環境での状況変化も反映
  - **参照情報源**：[inputファイル名または事例ページURL]

- **刺激（訴求軸）**: その段階でユーザーの態度変容を促すために効果的な価値訴求を15-20文字で整理
  - 各ペルソナの状況・業界・役職に応じた個別の刺激を設定
  - 同じ段階でもペルソナごとに異なる響く刺激を作成
  - ポジショニングの差別化ポイントを反映
  - 「その状況だからこそ出る刺激」を重視し、顧客の心理に深く刺さる表現を使用
  - **inputフォルダまたはトップページの訴求メッセージ、機能ページの差別化ポイントを活用**
  - **市場変化への対応**：将来の市場環境での訴求軸変化も考慮
  - **参照情報源**：[inputファイル名], [トップページURL], [機能ページURL]

### 参考項目（BtoB向け）：

- 社内調整プロセスの詳細
- 複数意思決定者の影響
- 長期的な検討プロセス
- ROIや業務効率化の考慮

### 参考項目（BtoC向け）：

- 感情的要素の影響
- ソーシャルプルーフの重要性
- 即時的な満足や体験の価値
- ブランドとの感情的なつながり

### 作成手順：

1. **前段階の情報を参照**：
   - ペルソナファイルから各ペルソナの詳細情報を取得
   - **特に「共通の購買プロセス」セクションから実際の顧客行動段階を抽出**
   - ポジショニングファイルから差別化ポイントを確認
   - ターゲティングファイルから市場特性を把握

0. **購買プロセス段階の決定**：
   - ペルソナファイルの「共通の購買プロセス」を基に、実際の顧客行動に合わせた段階を設定
   - 業界特性（BtoB/BtoC）、検討期間、意思決定の複雑さを考慮
   - 5-8段階で調整（例：認知→課題認識→情報収集→比較検討→社内調整→決定→導入→継続利用）

2. **inputフォルダ情報の活用**：
   - **input/自社/**: 実際の顧客ジャーニーデータ、タッチポイント情報、内部データ
   - **input/競合/**: 競合のカスタマージャーニー情報、タッチポイント戦略
   - **input/市場/**: 市場変化が顧客行動に与える影響、将来予測

3. **ウェブリサーチデータの補完活用**：
   - **case_studyページ**: 実際の顧客の検討プロセス、課題認識のタイミング
   - **homepageページ**: 各段階で響く訴求メッセージ
   - **featuresページ**: 検討段階で重要な差別化ポイント
   - **pricingページ**: 予算検討段階での価格感度、プラン選択
   - **serviceページ**: 情報収集段階で必要な詳細情報

3. **ペルソナ分のジャーニー作成**：
   - ペルソナファイルに記載されている全てのペルソナについて作成
   - 各ペルソナの業界・役職・課題に応じた状況を190-200文字で描写
   - **ペルソナファイルの「共通の購買プロセス」セクションから抽出した実際のプロセス段階を使用**（5-8段階で業界・サービス特性に応じて調整）

4. **ペルソナ別の刺激設計**：
   - 各段階で15-20文字の訴求軸を設定
   - ペルソナごとに異なる響く刺激を作成
   - 各ペルソナの業界・役職・課題に最適化された訴求軸
   - ポジショニングの差別化ポイントを反映
   - **ウェブリサーチから抽出した実際の訴求メッセージを活用**

### 🔍 ウェブリサーチの活用方法

#### ページタイプ別のカスタマージャーニー作成調査指針
- **事例ページ**: 実際の顧客の検討プロセス、各段階での行動パターン、決定要因
- **トップページ**: 各段階で響く核心メッセージ、ファーストインプレッション
- **サービス/製品ページ**: 情報収集段階での詳細情報、機能理解のプロセス
- **料金ページ**: 予算検討段階での価格比較、プラン選択のプロセス
- **機能ページ**: 比較検討段階での差別化ポイント、技術的優位性
- **会社概要ページ**: 信頼性確認段階での企業評価、実績確認

#### カスタマージャーニーの現実性向上
1. **実際の顧客行動ベース**: 事例ページの検討プロセスを参考
2. **段階別の具体的行動**: 各ページの情報から実際の情報収集行動を抽出
3. **現実的な検討期間**: 事例から実際の検討期間を反映
4. **具体的な決定要因**: 実際の選定理由を各段階の刺激に反映
5. **情報源の明確化**: 参照ページURLを記録してトレーサビリティを確保

### 出力ファイル：

1. **MDファイル**: `output/[ルートドメイン名]/ジャーニー_[企業名].md`
2. **CSVファイル**: `output/[ルートドメイン名]/ジャーニー_[企業名].csv`（上記の横型マトリックス形式）

この構造に基づいて、前段階で作成されたペルソナ全てに対応した現実的で詳細なカスタマージャーニーを作成してください。業界特性、製品/サービスの性質、ターゲットの特徴を十分に考慮し、**ウェブリサーチから抽出した実際の顧客行動パターン**を反映した実用的なジャーニーマップを構築してください。
