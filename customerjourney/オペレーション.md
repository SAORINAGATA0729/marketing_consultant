# ブランド戦略フレームワーク 自動実行オペレーション
# ウェブリサーチ対応版（BtoB・BtoC対応）

## 🚀 実行方法

以下の情報を入力するだけで、包括的なブランドマーケティング戦略レポートを自動生成します。
**対応範囲**: 企業、サービス、ツール、アプリ、ECサイト、店舗、個人ブランド等、あらゆるブランドに対応

---

## 📝 入力情報

### 1. 分析対象ブランド情報（必須）
**対象ブランドURL**：https://moltsinc.co.jp/
**ブランドタイプ**：企業サイト / サービスサイト / ECサイト / アプリ / 店舗 / 個人ブランド 等

### 2. 競合ブランド情報（必須）
**競合ブランドURL1**：https://ja.dev/
**競合ブランドURL2**：https://anagrams.jp/
**競合ブランドURL3**：https://sairu.co.jp/
**市場タイプ**：BtoB / BtoC / BtoB・BtoC混合

### 3. データ取得方法
**🔍 ウェブリサーチ分析**：
- 各ブランドのウェブサイト・プラットフォームを直接閲覧・分析
- 主要ページ（トップ、サービス、商品、事例、料金、概要等）の内容調査
- 公開情報に基づく事実ベースの分析
- 推測を排除した客観的データ収集

**📁 inputフォルダ情報の活用**：
- `input/自社/`：自社に関する追加情報（戦略、内部データ、非公開情報等）
- `input/競合/`：競合に関する追加情報（調査レポート、分析データ等）
- `input/市場/`：市場環境の変化情報（現在または未来の市場トレンド、規制変更等）

**🚨 重要な情報活用ルール**：

**❌ 絶対にやってはいけないこと**：
- 市場情報を自社情報として使用すること
- 推測で自社のサービス内容を創作すること
- inputフォルダの情報を間違った用途で使用すること

**✅ 正しい情報活用方法**：
1. **自社情報**: `input/自社/`を最優先、空の場合はウェブリサーチのみ
2. **競合情報**: `input/競合/`を最優先、空の場合はウェブリサーチのみ
3. **市場変化の影響**: `input/市場/`はターゲット・ペルソナの変化分析にのみ使用
4. **補完的活用**: inputに情報がない項目はウェブリサーチで事実ベース収集

**分析対象ページ（ブランドタイプ別）**：
- **企業・サービス**：トップページ、サービス詳細、事例・実績、料金情報、会社概要等
- **ECサイト・店舗**：トップページ、商品ページ、ブランドストーリー、レビュー、購入方法等
- **アプリ・ツール**：ランディングページ、機能説明、ユーザーレビュー、料金プラン等
- **個人ブランド**：プロフィール、実績、サービス内容、お客様の声等
- 出力先：`output/[ルートドメイン名]/` フォルダ内にMarkdownファイル

## ⚙️ 自動実行プロセス

上記の情報が提供されたら、以下のプロセスを自動的に実行します。

### 📊 実行ステップ

以下の指示に従って、同一親フォルダに格納された、プロンプトフォルダにある以下のファイルを読み取り、順に任務を遂行してください。また、指定された順番にて、展開してください。

**重要事項：**
1. **対象ブランドURLからルートドメイン名を抽出**（例：https://moltsinc.co.jp/ → moltsinc）
2. **outputフォルダ内に、ルートドメイン名のフォルダを作成**（例：output/moltsinc/）
3. **全ての出力ファイルを、このルートドメイン名フォルダ内に保存**
4. **各ステップで必ずファイルを出力してください**
5. **inputフォルダの情報を最優先で活用し、ウェブリサーチで補完してください**
6. **BtoB・BtoC・ブランドタイプに応じた適切な分析手法を使用してください**
7. **市場変化情報がある場合は、ターゲットの変化を考慮した分析を実施してください**

1. **ターゲティング.md**
   - **自社情報の収集**：`input/自社/`を最優先、空の場合はウェブリサーチのみ
   - **禁止事項**：市場情報を自社情報として使用することは絶対禁止
   - **市場変化の影響分析**：`input/市場/`はターゲットの変化分析にのみ使用
   - BtoB・BtoC・ブランドタイプに応じた分析
   - 出力：`output/[ルートドメイン名]/ターゲティング_[ブランド名].md`

2. **競合.md**
   - **競合情報の収集**：`input/競合/`を最優先、空の場合はウェブリサーチのみ
   - **禁止事項**：市場情報を競合情報として使用することは絶対禁止
   - 前提：ターゲティング結果を参照
   - **市場変化の影響分析**：`input/市場/`は競合環境の変化分析にのみ使用
   - 出力：`output/[ルートドメイン名]/競合調査_[ブランド名].md`

3. **ポジショニング.md**
   - 前提：ターゲティング結果と競合調査結果を参照
   - **情報統合**：自社・競合の実際の情報のみを統合
   - **市場変化の考慮**：`input/市場/`はポジショニング環境の変化分析にのみ使用
   - **禁止事項**：市場情報を自社・競合情報として使用することは絶対禁止
   - 出力：`output/[ルートドメイン名]/ポジショニング_[ブランド名].md`

4. **ペルソナ.md**
   - 前提：ターゲティング、ポジショニング結果を参照
   - **市場変化を反映したペルソナ**：`input/市場/`はペルソナの行動・ニーズ変化分析にのみ使用
   - **禁止事項**：市場情報を自社・競合情報として使用することは絶対禁止
   - BtoB・BtoC・ブランドタイプに応じたペルソナ開発
   - 出力：`output/[ルートドメイン名]/ペルソナ_[ブランド名].md`

5. **カスタマージャーニー.md**
   - 前提：全ての前段階の結果を参照
   - **市場変化を考慮したジャーニー**：`input/市場/`は顧客行動変化の分析にのみ使用
   - **禁止事項**：市場情報を自社・競合情報として使用することは絶対禁止
   - BtoB・BtoC・ブランドタイプに応じたジャーニー設計
   - 出力：
     - `output/[ルートドメイン名]/ジャーニー_[ブランド名].md`
     - `output/[ルートドメイン名]/ジャーニー_[ブランド名].csv`

### 📁 出力ファイル一覧

実行完了後、`output/[ルートドメイン名]/`フォルダに以下のファイルが生成されます：

**戦略分析ファイル（6ファイル）**
1. ターゲティング_[ブランド名].md

## 🆘 トラブルシューティング

### ウェブリサーチ関連

#### アクセス問題
- 対象サイトがメンテナンス中やアクセス制限をしている場合があります
- 一時的にサイトが表示されない場合は、時間をおいて再度アクセス
- 必要な情報が見つからない場合は、サイト内検索やサイトマップを活用

#### 情報不足の場合
- 企業の公式サイト以外に、プレスリリースやニュース記事も参考
- IRページや採用ページにも有用な情報が掲載されている場合があります
- SNSアカウントでの発信内容も補完情報として活用可能

### 分析精度を上げたい場合
- **複数ページの横断的調査を推奨**：より包括的で正確な情報収集
- 競合URLの選定を見直す
- より詳細な情報が掲載されたページを重点的に調査
