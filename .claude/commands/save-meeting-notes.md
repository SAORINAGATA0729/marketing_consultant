# 議事録保存スキル

Firefliesから議事録を取得し、適切なクライアントフォルダにリネームして保存する。

## 実行手順

### 1. Firefliesから最新の会議を取得
```
mcp__fireflies__fireflies_get_transcripts で最新10件を取得
```

ユーザーに会議を選択させる（番号で選択）。
引数で会議IDが指定されている場合はそれを使用。

### 2. 会議の詳細を取得
```
mcp__fireflies__fireflies_get_transcript_details で詳細取得
```

### 3. クライアント名を推測

**Step 3-1: クライアントフォルダを動的取得**
```
「クライアント/」フォルダの中身を一覧取得
→ 現在のクライアント一覧を動的に把握
```

**Step 3-2: 会議タイトルとマッチング**
会議タイトルに含まれるキーワードで自動判定：

| 検索キーワード | 備考 |
|--------------|------|
| フォルダ名そのまま | SHE, giftee, AeyeScan, Dsmart, tantan等 |
| 日本語読み | ギフティ→giftee, ミズテック等 |
| 略称・別名 | シーライクス→SHE等 |

特殊フォルダ：
| キーワード | フォルダ |
|-----------|---------|
| KAAAN, カーン | 社内/議事録/ |
| 寺倉 | 寺倉ナレッジ/会議一覧/ |
| 採用, 面談, 面接 | 採用/議事録/ |

**Step 3-3: マッチしない場合**
ユーザーに選択肢を提示：
```
クライアントが特定できませんでした。
どのフォルダに保存しますか？

1. SHE
2. giftee
3. AeyeScan
4. Dsmart
5. ミズテック
6. nikkenhomes
7. tantan
8. 新規フォルダを作成
```

### 4. ファイル名を生成
フォーマット: `YYMMDD_タイトル.md`

例:
- 会議日時: 2026-01-28
- タイトル: SHE週次定例
- → `260128_SHE週次定例.md`

タイトルは以下のルールで整形：
- クライアント名が重複する場合は削除（「SHE週次定例」→「週次定例」）
- 特殊文字を除去
- 長すぎる場合は30文字で切る

### 5. 議事録の内容を整形
以下のフォーマットで保存：

```markdown
# {タイトル}

**日時**: {YYYY年MM月DD日 HH:MM}
**参加者**: {参加者リスト}
**Fireflies URL**: {URL}

---

## サマリー
{AI生成サマリー}

## 議事録

{トランスクリプト本文}

---

## アクションアイテム
{抽出されたアクションアイテム}

## 次回予定
{あれば記載}
```

### 6. ファイルを保存
- フォルダが存在しない場合は作成
- 同名ファイルが存在する場合は確認

### 7. 完了報告
保存したファイルパスを報告。

---

## 使い方

```
/save-meeting-notes
```
→ 最新10件から選択

```
/save-meeting-notes {会議ID}
```
→ 指定した会議を直接保存

---

## 注意事項
- Firefliesに接続できない場合はエラーを表示
- クライアント名が特定できない場合は必ずユーザーに確認
- 寺倉さんの会議は `/terakura-sync` を使うことを推奨
