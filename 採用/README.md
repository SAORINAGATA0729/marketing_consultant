# 採用戦略フレームワーク 自動実行システム

## 📋 概要

このシステムは、企業の採用活動を支援するための戦略フレームワークを自動実行します。
既存の「カスタマージャーニー自動化システム」をベースに、採用専用に改修したシステムです。

## 🎯 主な機能

1. **ターゲティング分析** - 求職者層の分析
2. **ポジショニング分析** - 採用市場における企業のポジショニング
3. **ペルソナ開発** - 応募者のペルソナ作成（職種別）
4. **応募ジャーニー設計** - 応募プロセスの設計

## 📁 フォルダ構成

```
採用/
├── オペレーション.md          # 実行手順書
├── prompt/                    # 各ステップのプロンプト
│   ├── ターゲティング.md
│   ├── ポジショニング.md
│   ├── ペルソナ.md
│   └── 応募ジャーニー.md
├── input/                     # 追加情報
│   ├── 自社/                 # 自社の採用情報
│   └── 市場/                 # 採用市場の変化情報
└── output/                    # 生成結果
    └── [ルートドメイン名]/
        ├── ターゲティング_[企業名].md
        ├── ポジショニング_[企業名].md
        ├── ペルソナ_[企業名].md
        ├── 応募ジャーニー_[企業名].md
        └── 応募ジャーニー_[企業名].csv
```

## 🚀 使い方

### Step 1: 入力情報の準備

**オペレーション.mdの「入力情報」セクションに以下を記入：**

1. **分析対象企業情報**
   - 対象企業URL（例：`https://kaaan.jp/`）
   - 企業タイプ（企業サイト/サービスサイト等）

2. **募集職種情報**
   - 募集職種1（例：エンジニア）
   - 募集職種2（例：PDM）
   - 募集職種3（例：コンサルタント）
   - ※職種は3つまで指定可能（1つでも可）

### Step 2: inputフォルダの確認（任意）

- `input/自社/`に追加情報があれば入れる
- `input/市場/`に採用市場の変化情報があれば入れる

### Step 3: 実行開始

**オペレーション.mdをCursor AIで開いて、「実行してください」と入力**

---

## ⚙️ 自動実行プロセス

AIが以下の順で自動実行します：

1. **ターゲティング分析** → ファイル出力
2. **ポジショニング分析** → ファイル出力
3. **ペルソナ開発** → ファイル出力（職種ごとに1-2名）
4. **応募ジャーニー設計** → ファイル出力（MD + CSV）

**各ステップでファイルが生成されるので、確認しながら進めます。**

## 📊 出力ファイル

実行完了後、`output/[ルートドメイン名]/`フォルダに以下のファイルが生成されます：

1. **ターゲティング_[企業名].md** - 求職者層の分析
2. **ポジショニング_[企業名].md** - 採用市場における企業のポジショニング
3. **ペルソナ_[企業名].md** - 応募者のペルソナ（職種別）
4. **応募ジャーニー_[企業名].md** - 応募プロセスの詳細
5. **応募ジャーニー_[企業名].csv** - 応募プロセスのマトリックス形式

## 🔍 既存システムとの違い

### カスタマージャーニー自動化システム（既存）
- **ターゲット**: 顧客（サービス利用者）
- **ペルソナ**: 購買ストーリー
- **ジャーニー**: 購買プロセス
- **競合調査**: あり（3社）

### 採用戦略フレームワーク（本システム）
- **ターゲット**: 求職者・応募者
- **ペルソナ**: 応募ストーリー
- **ジャーニー**: 応募プロセス
- **競合調査**: なし（競合が特にいないため削除）

## ⚠️ 重要な注意事項

### 情報活用ルール

**❌ 絶対にやってはいけないこと**：
- 市場情報を自社情報として使用すること
- 推測で自社の採用内容を創作すること
- inputフォルダの情報を間違った用途で使用すること

**✅ 正しい情報活用方法**：
1. **自社情報**: `input/自社/`を最優先、空の場合はウェブリサーチのみ
2. **市場変化情報**: `input/市場/`はターゲット（求職者）・ペルソナの変化分析にのみ使用
3. **補完的活用**: inputに情報がない項目はウェブリサーチで事実ベース収集

## 📝 実行前チェックリスト

- [ ] 企業URLは正しく入力されているか
- [ ] 募集職種が入力されているか（最低1つ）
- [ ] URLはアクセス可能か（404エラーでないか）
- [ ] inputフォルダに追加情報があるか確認（任意）
- [ ] `input/自社/`：自社情報として使用
- [ ] `input/市場/`：ターゲット（求職者）・ペルソナの変化分析にのみ使用

## 🆘 トラブルシューティング

### ウェブリサーチ関連

#### アクセス問題
- 対象サイトがメンテナンス中やアクセス制限をしている場合があります
- 一時的にサイトが表示されない場合は、時間をおいて再度アクセス

#### 情報不足の場合
- 企業の公式サイト以外に、プレスリリースやニュース記事も参考
- 採用サイトやSNSアカウントでの発信内容も補完情報として活用可能
- 社員インタビューやブログにも有用な情報が掲載されている場合があります
