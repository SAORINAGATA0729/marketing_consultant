# 採用戦略フレームワーク 自動実行オペレーション
# ウェブリサーチ対応版

## 🚀 実行方法

以下の情報を入力するだけで、包括的な採用戦略レポートを自動生成します。
**対応範囲**: 企業の採用活動全般に対応

---

## 📝 入力情報

### 1. 分析対象企業情報（必須）
**対象企業URL**：https://kaaan.jp/
**企業タイプ**：企業サイト / サービスサイト / ECサイト / アプリ / 店舗 等

### 2. 募集職種情報（必須）
**募集職種1**：エンジニア
**募集職種2**：PDM（プロダクトマネージャー）
**募集職種3**：コンサルタント
※職種は3つまで指定可能（1つでも可）

### 3. データ取得方法
**🔍 ウェブリサーチ分析**：
- 企業のウェブサイト・採用サイトを直接閲覧・分析
- 主要ページ（トップ、採用ページ、会社概要、サービス等）の内容調査
- 公開情報に基づく事実ベースの分析
- 推測を排除した客観的データ収集

**📁 inputフォルダ情報の活用**：
- `input/自社/`：自社に関する追加情報（採用戦略、内部データ、非公開情報等）
- `input/市場/`：採用市場環境の変化情報（現在または未来の採用市場トレンド、労働市場の変化等）

**🚨 重要な情報活用ルール**：

**❌ 絶対にやってはいけないこと**：
- 市場情報を自社情報として使用すること
- 推測で自社の採用内容を創作すること
- inputフォルダの情報を間違った用途で使用すること

**✅ 正しい情報活用方法**：
1. **自社情報**: `input/自社/`を最優先、空の場合はウェブリサーチのみ
2. **市場変化の影響**: `input/市場/`はターゲット（求職者）・ペルソナの変化分析にのみ使用
3. **補完的活用**: inputに情報がない項目はウェブリサーチで事実ベース収集

**分析対象ページ**：
- **企業サイト**：トップページ、採用ページ、会社概要、サービス詳細、メンバー紹介等
- **採用サイト**：募集要項、職種別ページ、社員インタビュー、企業文化等
- 出力先：`output/[ルートドメイン名]/` フォルダ内にMarkdownファイル

---

## ⚙️ 自動実行プロセス

上記の情報が提供されたら、以下のプロセスを自動的に実行します。

### 📊 実行ステップ

以下の指示に従って、同一親フォルダに格納された、プロンプトフォルダにある以下のファイルを読み取り、順に任務を遂行してください。また、指定された順番にて、展開してください。

**重要事項：**
1. **対象企業URLからルートドメイン名を抽出**（例：https://kaaan.jp/ → kaaan）
2. **outputフォルダ内に、ルートドメイン名のフォルダを作成**（例：output/kaaan/）
3. **全ての出力ファイルを、このルートドメイン名フォルダ内に保存**
4. **各ステップで必ずファイルを出力してください**
5. **inputフォルダの情報を最優先で活用し、ウェブリサーチで補完してください**
6. **市場変化情報がある場合は、ターゲット（求職者）の変化を考慮した分析を実施してください**

1. **ターゲティング.md**
   - **自社情報の収集**：`input/自社/`を最優先、空の場合はウェブリサーチのみ
   - **禁止事項**：市場情報を自社情報として使用することは絶対禁止
   - **市場変化の影響分析**：`input/市場/`はターゲット（求職者）の変化分析にのみ使用
   - 出力：`output/[ルートドメイン名]/ターゲティング_[企業名].md`

2. **ポジショニング.md**
   - 前提：ターゲティング結果を参照
   - **情報統合**：自社の実際の情報のみを統合
   - **市場変化の考慮**：`input/市場/`はポジショニング環境の変化分析にのみ使用
   - **禁止事項**：市場情報を自社情報として使用することは絶対禁止
   - 出力：`output/[ルートドメイン名]/ポジショニング_[企業名].md`

3. **ペルソナ.md**
   - 前提：ターゲティング、ポジショニング結果を参照
   - **市場変化を反映したペルソナ**：`input/市場/`はペルソナ（応募者）の行動・ニーズ変化分析にのみ使用
   - **禁止事項**：市場情報を自社情報として使用することは絶対禁止
   - 職種別のペルソナ開発（募集職種ごとに1-2名のペルソナを作成）
   - 出力：`output/[ルートドメイン名]/ペルソナ_[企業名].md`

4. **応募ジャーニー.md**
   - 前提：全ての前段階の結果を参照
   - **市場変化を考慮したジャーニー**：`input/市場/`は応募者行動変化の分析にのみ使用
   - **禁止事項**：市場情報を自社情報として使用することは絶対禁止
   - 職種別の応募プロセス設計
   - 出力：
     - `output/[ルートドメイン名]/応募ジャーニー_[企業名].md`
     - `output/[ルートドメイン名]/応募ジャーニー_[企業名].csv`

### 📁 出力ファイル一覧

実行完了後、`output/[ルートドメイン名]/`フォルダに以下のファイルが生成されます：

**戦略分析ファイル（4ファイル）**
1. ターゲティング_[企業名].md
2. ポジショニング_[企業名].md
3. ペルソナ_[企業名].md
4. 応募ジャーニー_[企業名].md
5. 応募ジャーニー_[企業名].csv

**ウェブリサーチ分析ファイル**
- web_research_[企業ドメイン]_[日時].md（対象企業の分析データ）
- research_summary_[日時].md（統合分析レポート）

**例）kaaan.jpの場合**
```
output/kaaan/
├── ターゲティング_KAAAN.md
├── ポジショニング_KAAAN.md
├── ペルソナ_KAAAN.md
├── 応募ジャーニー_KAAAN.md
├── 応募ジャーニー_KAAAN.csv
├── web_research_kaaan_20250928_143022.md
└── research_summary_20250928_143155.md
```

## 🔄 実行開始

**以下の手順で実行を開始します：**

1. 上記の入力情報セクションにURLと募集職種を記入
2. **inputフォルダの確認**：`input/自社/`、`input/市場/`の内容を確認
3. 企業のウェブサイトを直接調査・分析（inputで不足する情報を補完）
4. このファイル全体を実行

システムが自動的に各プロンプトを順次実行し、inputフォルダの情報を最優先に活用した包括的な採用戦略レポートを生成します。

**📋 実行前の確認事項**：
- [ ] inputフォルダに追加情報があるか確認
- [ ] `input/自社/`：自社情報として使用
- [ ] `input/市場/`：ターゲット（求職者）・ペルソナ・応募者行動の変化分析にのみ使用
- [ ] **絶対禁止**：市場情報を自社情報として使用しない
- [ ] **絶対禁止**：推測で自社の採用内容を創作しない

## 🆘 トラブルシューティング

### ウェブリサーチ関連

#### アクセス問題
- 対象サイトがメンテナンス中やアクセス制限をしている場合があります
- 一時的にサイトが表示されない場合は、時間をおいて再度アクセス
- 必要な情報が見つからない場合は、サイト内検索やサイトマップを活用

#### 情報不足の場合
- 企業の公式サイト以外に、プレスリリースやニュース記事も参考
- 採用サイトやSNSアカウントでの発信内容も補完情報として活用可能
- 社員インタビューやブログにも有用な情報が掲載されている場合があります

### 分析精度を上げたい場合
- **複数ページの横断的調査を推奨**：より包括的で正確な情報収集
- より詳細な情報が掲載されたページを重点的に調査
