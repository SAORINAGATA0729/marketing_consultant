## KPIシートの仕組み

### Google Apps Script による自動生成

**特徴:**
- 手動でシート構造を作る必要なし
- スクリプト実行で3つのシートが自動生成される
- 数式、書式、条件付き書式がすべて自動設定

**生成されるシート:**
1. **📊 サマリー** - 経営層向けダッシュボード
2. **📝 週次入力** - 週次実績入力シート
3. **👥 リファラル** - リファラル実績管理

**データフロー:**
```
週次入力シート（手入力）
    ↓ 自動参照
サマリーシート（自動計算）
    ↓
経営判断・改善施策
```

### 自動計算される指標

- 達成率（実績 ÷ 目標）
- 返信率（返信 ÷ スカウト）
- 面談化率（面談 ÷ 返信）
- 面談単価（費用 ÷ 面談数）
- 採用単価（費用 ÷ 採用数）

### 条件付き書式（色分け）

| 達成率 | 色 |
|--------|-----|
| 100%以上 | 緑 |
| 50-99% | 黄色 |
| 50%未満 | 赤 |

## 運用フロー

### 週次定例（毎週月曜 11:00-11:30）

**参加者:** 永田、松原

**アジェンダ:**
1. 今週の進捗確認（5分）
2. 来週やること（5分）
3. ブロッカー・相談事項（5分）

**確認シート:**
- KPIシート: https://docs.google.com/spreadsheets/d/1jLGK_qUW-89YUOMDW_QuNWBanmHTT7U_72UBcB_11Fo/
- 採用管理シート: https://docs.google.com/spreadsheets/d/1Xs3__LaMXnMCMxEv6B6Qq9FgBI1cG0V6-l91XbhlYPE/

**タスク管理:**
- 採用管理シートの「タスク進捗」タブで管理
- ステータス: ❌未着手 / 🔄進行中 / ✅完了（プルダウン）
- 「今週やること」にチェックを入れる → 定例後にSlackに自動投稿

### 週次KPI入力（毎週月曜 5分）

**担当:** 採用担当

**やること:**
1. 📝 週次入力シートを開く
2. 該当週の行に前週実績を入力
   - スカウト送信数
   - 返信数
   - 面談数（Wantedly / リファラル）
   - 採用数（あれば）
3. 📊 サマリーシートで自動計算を確認

**目標値（週次）:**
- スカウト送信: 25件
- 返信: 12-13件
- 面談: 12-13件

### 月次（月初3営業日以内 15分）

**担当:** 採用責任者

**やること:**
1. 達成率・転換率の確認
2. ボトルネック特定（50%未満の項目）
3. 改善施策の検討
4. 次月計画の調整

### 四半期（3Q末、4Q末 30分）

**担当:** 採用責任者

**やること:**
1. 費用対効果分析
2. チャネル別効果測定
3. 次四半期の予算・施策見直し

## Apps Script について

### 1. 採用定例カレンダー登録（採用定例カレンダー登録.gs）

**機能:**
- 週次定例（毎週月曜11:00-11:30）をGoogleカレンダーに12週分登録
- 参加者に招待メール送信
- 説明欄にアジェンダ・シートリンクを自動挿入

**関数:**
- `createRecurringMeeting()` - 定例を登録
- `deleteRecurringMeetings()` - 登録済み定例を削除

**実行方法:**
1. Google Apps Script（script.google.com）で新規プロジェクト作成
2. コードをコピペ
3. `createRecurringMeeting()` を実行
4. 初回は権限承認が必要

### 2. タスク管理シート生成（採用タスク管理シート生成.gs）

**機能:**
- 採用管理シートに「タスク進捗」タブを追加
- ステータス（プルダウン）、今週やること（チェックボックス）、メモ列
- ステータスで行の色が自動変更（緑/黄/赤）
- 毎週月曜11:30にSlack自動通知（要Webhook URL設定）

**関数:**
- `createTaskSheet()` - タスク進捗シートを生成
- `sendWeeklySlackNotification()` - Slack通知送信
- `testSlackNotification()` - Slack通知テスト

**実行方法:**
1. 採用管理シートを開く
2. 拡張機能 > Apps Script
3. コードをコピペ
4. `createTaskSheet()` を実行

**Slack通知設定:**
1. Slack Webhook URLを取得
2. コード内の `SLACK_WEBHOOK_URL` を置き換え
3. トリガー設定: 毎週月曜11:30に `sendWeeklySlackNotification` を実行

### 3. Wantedly自動化システム設計図（Wantedly自動化システム設計図.gs）

**機能:**
- 採用管理シートに「システム全体像」タブを追加
- Wantedly自動化システムのフローチャートを可視化
- フェーズ1（スカウト〜面談設定）とフェーズ2（議事録連携）を図示

**関数:**
- `createSystemOverviewSheet()` - システム全体像シートを生成

**フローチャート内容:**
- フェーズ1: スクリーニング → AIチェック → スプシ記入 → メッセージ生成 → 送付 → 返信検知 → 面談設定
- ポジション分岐: コンサル → 田島/永田、PdM → そめひこ
- フェーズ2: 面談実施 → Fireflies議事録 → カレンダー連携

**凡例:**
- 🤖 緑 = AI処理
- 👤 黄 = 人間作業
- 青 = データ/状態

**実行方法:**
1. 採用管理シートを開く
2. 拡張機能 > Apps Script
3. コードをコピペ
4. `createSystemOverviewSheet()` を実行

### 3. KPIシート自動生成（KAAAN採用KPI自動生成.gs）

**関数:**
- `createKAANRecruitingKPI()` - メイン関数
- `createSummarySheet()` - サマリーシート生成
- `createWeeklySheet()` - 週次入力シート生成
- `createReferralSheet()` - リファラルシート生成

### 再生成について

**注意:** スクリプトを再実行すると、既存シートが上書きされます。

シート構造を変更したい場合:
1. Apps Script のコードを修正
2. 再実行して新しいシート生成
3. 既存データを手動でコピペ

## トラブルシューティング

### Q1: スプレッドシートが見つからない

**対処:**
- Apps Script 実行ログでURLを確認
- Google Drive で「KAAAN 採用KPIシート」を検索

### Q2: 数式が壊れた

**対処:**
- Apps Script を再実行して新しいシート生成
- データを手動で移行

### Q3: 週次入力を忘れた

**対処:**
- 気づいた時点で入力
- Wantedly管理画面で過去データを確認

### Q4: 目標達成率が低い

**対処:**
1. どのステップで詰まっているか特定（返信率？面談化率？）
2. ボトルネックに応じた施策を実施
   - 返信率低い → スカウト文面改善
   - 面談化率低い → 返信後のフォロー強化
