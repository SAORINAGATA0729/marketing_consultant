# Wantedly候補者管理ワークフロー

Wantedlyエージェント機能を使って候補者をピックアップし、スクリーニングから送付まで自動化するワークフローです。

**このワークフローはプロンプトチェーンで構成されており、一連のやり取りの中で順番に進めていきます。**

## 🚀 クイックスタート（推奨）

**マスタープロンプト「候補者サーチOPs」を使用すると、すべてのステップが自動的に実行されます。**

CursorのComposerで以下のように実行してください：

```
候補者サーチOPsを実行してください。
```

または：

```
prompts/00_候補者サーチOPs.md の内容を読み込んで実行してください。
```

**実行すると**:
1. Wantedlyページが自動的に開かれます
2. エージェント起動の手順が表示されます
3. ユーザーがエージェントで検索を完了すると、以降のステップが自動的に実行されます

---

## 📋 手動実行（個別ステップ実行）

各ステップを個別に実行したい場合は、以下の手順に従ってください。

## フォルダ構造

```
Wantedly候補者管理/
├── 候補者フォルダ/      # ピックアップされた候補者のmdファイル
├── OKフォルダ/          # スクリーニングでOKと判断された候補者
├── NGフォルダ/          # スクリーニングでNGと判断された候補者
├── 送付文面/            # 作成された送付文面
├── prompts/             # 各ステップ用のプロンプト
├── scripts/             # 自動化スクリプト
└── README.md           # このファイル
```

## プロンプトチェーンの流れ

このワークフローは、以下の4つのステップで構成されたプロンプトチェーンです。
**各ステップは順番に実行し、ユーザー確認を経て次のステップへ進みます。**

```
ステップ1: 候補者ピックアップ
    ↓ (ユーザー確認)
ステップ2: 候補者情報md化
    ↓ (ユーザー確認)
ステップ3: 候補者判断
    ↓ (ユーザー確認)
ステップ4: 送付文面作成
    ↓
完了（Wantedlyに送付）
```

---

## クイックスタート

### ステップ1: 候補者ピックアップ

**前提条件**: ユーザーがWantedlyエージェントを起動し、条件設定を完了し、候補者の結果が表示されている状態

CursorのComposerで以下のプロンプトを使用：

```
prompts/01_候補者ピックアップ.md の内容を読み込んで実行してください。

候補者の結果が表示されました。
```

**このステップで行うこと**:
- 候補者一覧からプロフィールURLを抽出
- ユーザーに「誰をピックアップするか」「何人ピックアップするか」を確認
- 確定した候補者URLリストを準備

**完了後**: ユーザー確認を経て、次のステップへ進む

---

### ステップ2: 候補者情報md化

**前提条件**: ステップ1で候補者URLリストが確定している

CursorのComposerで以下のプロンプトを使用：

```
prompts/02_候補者情報md化.md の内容を読み込んで実行してください。

以下の候補者URLリストから各候補者のプロフィールを開いて、情報をmdファイルに変換してください。

[ここにステップ1で確定した候補者URLリストを貼り付け]
```

**このステップで行うこと**:
- 各候補者のプロフィールページを開く
- 情報を抽出してmdファイルに変換
- `候補者フォルダ/[氏名]_[YYYYMMDD].md` として保存

**完了後**: ユーザー確認を経て、次のステップへ進む

---

### ステップ3: 候補者判断

**前提条件**: ステップ2で候補者フォルダ内にmdファイルが作成されている

CursorのComposerで以下のプロンプトを使用：

```
prompts/03_候補者判断.md の内容を読み込んで実行してください。

候補者フォルダ内のmdファイルを読み込み、各候補者をOK/NGに判断してください。
```

**このステップで行うこと**:
- 各候補者の実行力・推進力を分析
- OK/NGを判断
- mdファイルの「候補者分析結果」「スクリーニング結果」セクションを更新

**完了後**: 
- OKと判断された候補者をOKフォルダへ移動（手動または`scripts/move_candidates.py`）
- ユーザー確認を経て、次のステップへ進む

**フォルダ移動の方法**:
```bash
python scripts/move_candidates.py
```

または、CursorのComposerで：
```
scripts/move_candidates.py を実行して、候補者フォルダ内のmdファイルをOK/NGフォルダへ移動してください。
```

---

### ステップ4: 送付文面作成

**前提条件**: ステップ3でOKと判断された候補者がOKフォルダに移動している

CursorのComposerで以下のプロンプトを使用：

```
prompts/04_送付文面作成.md の内容を読み込んで実行してください。

OKフォルダ内の候補者に対して送付文面を作成してください。
```

**このステップで行うこと**:
- OKフォルダ内の候補者の分析結果を確認
- THE MOLTSを主語としたスカウト文面を作成
- `送付文面/[氏名]_送付文面_[YYYYMMDD].md` として保存

**完了後**: プロンプトチェーン完了。作成された送付文面をWantedlyにコピペして送付

---

### 送付

作成された送付文面をWantedlyのスカウトメッセージ欄にコピペして送付します。

## 重要な注意事項

### ユーザーが行うこと

1. **Wantedlyエージェントの起動と条件設定**
   - Wantedly管理画面（https://admin.wantedly.com/scouts/users/）にアクセス
   - エージェントモードを起動（画面上部、「絞り込み」の右側にある紫色のアイコンをクリック）
   - 条件設定（職種、スキル、経験年数など）を完了
   - 候補者の結果が表示されたら、AIに「候補者の結果が表示されました」と伝える

2. **各ステップでの確認**
   - 各ステップが完了したら、内容を確認
   - 「次のステップに進んでください」と確認してから次へ進む

3. **OKフォルダへの移動**
   - ステップ3でOKと判断された候補者をOKフォルダへ移動
   - 手動で移動するか、`scripts/move_candidates.py`を実行

### AIが行うこと

1. **候補者ピックアップ**
   - ユーザーがエージェントを起動した後の状態から開始
   - 候補者一覧からプロフィールURLを抽出
   - 「誰をピックアップするか」「何人ピックアップするか」をユーザーに確認

2. **候補者情報md化**
   - 確定した候補者URLリストから各候補者のプロフィールを開く
   - 情報を抽出してmdファイルに変換

3. **候補者判断**
   - 候補者フォルダ内のmdファイルを分析
   - OK/NGを判断

4. **送付文面作成**
   - OKフォルダ内の候補者に対して送付文面を作成

### エージェントの制約

- **エージェントは1回で最大10人の候補者を挙げる**
- AIは「誰をピックアップするか」「何人ピックアップするか」をユーザーに確認して実行

## プロンプト一覧

各ステップで使用するプロンプトは以下のファイルを参照してください：

- `prompts/00_候補者サーチOPs.md` - **マスタープロンプト（推奨）** - 全ステップを自動実行
- `prompts/01_候補者ピックアップ.md` - 候補者ピックアップ用プロンプト
- `prompts/02_候補者情報md化.md` - プロフィール情報をmdに変換するプロンプト
- `prompts/03_候補者判断.md` - FBオペレーション用判断プロンプト
- `prompts/04_送付文面作成.md` - 送付文面作成用プロンプト

## スクリプト一覧

- `scripts/move_candidates.py` - OK/NG判断に基づいてファイルを自動移動

詳細は `scripts/README.md` を参照してください。

## プロンプトチェーンの使い方

### 基本的な流れ

1. **最初のステップから開始**
   ```
   prompts/01_候補者ピックアップ.md の内容を読み込んで実行してください。
   候補者の結果が表示されました。
   ```

2. **各ステップでユーザー確認**
   - AIが各ステップを完了したら、内容を確認
   - 「次のステップに進んでください」と伝える

3. **次のステップのプロンプトを読み込む**
   - 次のステップのプロンプトファイルを読み込む
   - 前のステップで準備されたデータを使用

4. **最後まで繰り返す**
   - ステップ4まで完了すると、プロンプトチェーンが完了

### 各プロンプトの特徴

各プロンプトには以下が含まれています：
- **このプロンプトの位置づけ**: プロンプトチェーン内での位置
- **前提条件**: 実行前に必要な条件
- **実行手順**: 具体的な実行方法
- **次のステップへの移行条件**: 次のステップへ進むための条件

---

## 注意事項

- **各ステップは順番に実行してください**（プロンプトチェーンの順序を守る）
- **各ステップでユーザー確認を取ってから次へ進む**（確認なしで進まない）
- 候補者情報のmd化は時間がかかる場合があります（1人あたり30秒〜1分程度）
- 候補者判断の基準は、実際の採用要件に合わせて調整してください
- 送付文面は候補者ごとに個別に作成してください（コピペは避ける）
- **Wantedlyエージェントの起動や条件設定はユーザーが行う**（AIは行わない）
