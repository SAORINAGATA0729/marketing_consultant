## コンテンツ方針（2026/1/20 MTGで決定）

### 基本方針
1. **機能ページをしっかり作る** - 1機能1ページで切り出す
2. **深掘り記事を作成** - 「使ってみたらこんなことができた！」系
3. **SEOを強化する** - ドメインパワーの底上げ
4. **FAQは優先度高くないが、やらないと決める理由もない**

### なぜ1機能1ページか（LLMO観点）
- 機能一覧ページだけでは、LLMが「その機能はない」と誤認するリスク
- 詳細ページを作ることで特定クエリに検索順位がつきやすい
- 結果としてAIの回答にも引用されやすくなる

## 作成すべき機能ページ

| 機能 | 訴求ポイント |
|------|-------------|
| **依頼URL機能** | アカウント不要で外部の人もタスク追加可能 |
| **タスク内チャット機能** | Slackで探す時間ゼロ（12秒 vs 30秒の差） |
| **グループとルール設定** | プロジェクトごとにルール固定、説明コスト削減 |
| **タスク完了確認・アラート** | 未対応者への催促コスト削減 |
| **チームステータス把握** | 誰が何を持っているか一目でわかる |
