# tantan サイトマップ：現状 vs あるべき姿

**作成日**: 2026年1月20日
**目的**: 「1機能1ページ」方針に基づき、新規作成すべきページを明確化

---

## 現状のサイト構造

```
tantan-task.com/
├── / (TOP・LP) ← 全てここに集約
│   ├── #features（機能一覧セクション）
│   └── #pricing（料金セクション）
├── /auth/signin（ログイン/サインアップ）
└── （他ページなし）
```

### 現状の問題点
- **機能が1ページに羅列** → LLMが「その機能はない」と誤認するリスク
- **独立した機能ページがない** → 特定クエリで検索順位がつきにくい
- **SEO対策が弱い** → ドメインパワーが上がりにくい

## 補足：機能ページの構成案

各機能ページに含めるべき要素：

1. **機能名・キャッチコピー**
2. **解決する課題**（誰のどんな痛みを解決するか）
3. **機能の詳細説明**（スクリーンショット付き）
4. **ユースケース**（具体的な利用シーン）
5. **導入効果**（Before/After）
6. **CTA**（無料で始める）
