# ペルソナ開発レポート：タンタン v2

## エグゼクティブサマリー

### 作成したペルソナ（優先度順）
1. **「忙殺されるマーケティングマネージャー」** - クライアントワークに追われる支援会社のマネージャー
2. **「関わる人が多い事業会社リーダー」** - 業務委託やチームを巻き込んでサービスを作るスタートアップ責任者
3. **「複数クライアントを抱えるWeb系フリーランス」** - 依頼の整理に時間を取られる個人事業主

### 各ペルソナの主要な課題
- ペルソナ1：クライアントワークが多く、Slack・メール・口頭からの依頼が散らばり、タスク登録だけで時間を消費
- ペルソナ2：業務委託、社員、パートナーなど関わる人が多く、依頼・進捗管理が属人化、「言った言わない」問題
- ペルソナ3：クライアントごとに異なる依頼方法、情報の抜け漏れが発生

### 購買に至る主要な動機
- 「タスクを拾い集める」作業から解放されたい
- シンプルで学習コストの低いツールを求めている
- チーム・関係者との連携をスムーズにしたい

### 3ペルソナ共通のポイント
**「人との関わりが多い人」**がタンタンのコアターゲット

※参照情報源：https://www.tantan-task.com/

## ペルソナ1：忙殺されるマーケティングマネージャー

### 基本設定
- **業界**：IT/Web、マーケティング支援会社
  - クライアントへのマーケティング施策支援が主業務
  - 複数クライアントを同時に担当
- **企業規模**：従業員30〜100名程度の中小企業
- **役職**：マーケティング部長 / マネージャー
- **部門**：マーケティング部門
- **関わる人**：チームメンバー5名、複数クライアント、外部パートナー（ライター、デザイナー等）

### なぜこの人がターゲットか
- **クライアントワークが多い** = 依頼元が多様
- 社内からもクライアントからも依頼が来る
- チャネルがバラバラ（Slack、メール、MTGでの口頭依頼）
- 「自分中心で管理したいが、周りと繋がる必要がある」というタンタンの価値にマッチ

### 購買ストーリー（約1,200文字）

#### 1. 日常の仕事と状況

私はIT系マーケティング支援会社でマーケティングマネージャーとして働いています。日々の業務はチームメンバー5名の統括と、**複数クライアントのマーケティング施策の進行管理**。主にキャンペーン企画、コンテンツ制作の指示、数値分析レポートの作成を担当しています。

最近の仕事の状況は、正直言って「カオス」です。**社内メンバーからの相談、クライアントからの急な依頼、外部ライターへの発注管理**...。Slackでは複数のチャンネルから常に通知が飛んできて、メールでは外部パートナーからの依頼、社内ミーティングでは口頭での「これお願い」が次々と。1日の終わりにふと気づくと、本当にやるべき仕事より「タスクを整理する」ことに2時間も使っていたりします。

※参照ページ：https://www.tantan-task.com/（「増え続けるツール、散らばる会話」の課題提示）

#### 2. 直面している課題

実は、ここ数ヶ月「タスクの抜け漏れ」に悩んでいます。

きっかけは先月のことでした。重要クライアントへの提案資料の提出が1日遅れてしまったのです。クライアントからSlackで依頼されていたのを見落としていました。上司からは「なぜ漏れたのか」と問われ、正直に「Slackの通知が多すぎて見落とした」と答えました。でも、それは言い訳でしかありません。

社内では「佐藤さんはマルチタスクが得意」と思われていますが、実際は毎日必死で**クライアント別のSlackチャンネル、社内チャンネル、メール**を遡り、自分でタスク登録しています。特にクライアントからの「これお願いできますか？」という依頼は、いつも自分でメモに書き写している。この「名もなき作業」だけで、週に5時間は消えている気がします。

※参照ページ：https://www.tantan-task.com/（「その『名もなき作業』に、さよならを」のメッセージ）

#### 3. 解決への動機と行動

この状況を打破するため、「タスク管理ツール」で情報収集を始めました。

最初はAsanaやTrelloを試しました。機能は充実しているのですが、結局「自分でタスクを登録する」という作業は変わらない。カンバンが見やすくなっても、クライアントからのSlackメッセージを自分で転記する手間は残ったままでした。

同じ業界（マーケ支援会社）の知人に相談したところ、「タンタンっていうツール、試してみた？」と教えてもらいました。「URLを共有するだけで、相手が直接タスクを追加してくれる」という説明を聞いて、「それだ！」と思いました。

※参照ページ：https://www.tantan-task.com/（「URLを共有するだけで、全ての依頼が自動でリストに整列」）

#### 4. サービス検討のプロセス

いくつかのサービスを比較検討する中で、重視したのは以下の点です：

1. **タスク登録の手間が本当に減るか**：自分で転記する作業をなくしたい
2. **シンプルで迷わないか**：複雑な設定は避けたい
3. **チームとクライアント両方で使えるか**：社内外どちらとも連携したい

タンタンを調べる中で、「タスクルール設定」機能に魅力を感じました。「期限は必須」「参考URLを添付して」など、私がいつも口頭で言っていたことを、システムが代わりに指示してくれる。これならクライアントへの細かい指示出しも不要です。

※参照ページ：https://www.tantan-task.com/（タスクルール設定機能）

#### 5. 意思決定の理由

最終的にタンタンを選ぶ決め手となったのは、「発想の転換」でした。

他のツールは「自分がタスクを管理する」前提。でもタンタンは「周りにも追加してもらう」という発想。この違いが、**クライアントワークが多い私の課題**を根本から解決してくれると確信しました。

導入後は、チームメンバーや外部パートナー、さらにはクライアントにもURLを共有し、直接タスクを追加してもらう運用を始めたいと考えています。「タスクを拾い集める」時間を、「タスクをこなす」時間に変えたい。それが私の目標です。

※参照ページ：https://www.tantan-task.com/（「拾う時間を、こなす時間へ」のメッセージ）

### このペルソナへのアプローチ指針
- **響くメッセージ**：「クライアントからの依頼も、社内からの依頼も、1つの場所に」
- **重要なタイミング**：タスク漏れによるクライアントトラブル発生後、年度初めの業務改善検討時
- **提供すべき情報**：クライアント別URL発行、チーム連携機能の具体的な使い方、導入の容易さ

## ペルソナ2：関わる人が多い事業会社リーダー

### 基本設定
- **業界**：IT/Web、SaaS、スタートアップ（事業会社）
- **企業規模**：従業員3〜30名程度のスタートアップ / 小規模事業会社
- **役職**：COO / 事業責任者 / チームリーダー / 創業メンバー
- **部門**：事業開発 / プロダクト開発
- **関わる人**：
  - 社員（3〜10名）
  - 業務委託（エンジニア、デザイナー、マーケターなど）
  - パートナー企業
  - 副業メンバー

### なぜこの人がターゲットか
- **関わる人が多い** = 依頼・受け渡しが頻発
- 社員だけでなく、**業務委託・パートナー・副業メンバー**など雇用形態が多様
- 少人数で多くのことをこなすため、コミュニケーション効率が命
- 「言った言わない」問題が発生しやすい環境
- でもAsanaやBacklogは大げさすぎる / 全員に使い方を教える時間がない

### 購買ストーリー（約1,200文字）

#### 1. 日常の仕事と状況

私はSaaSスタートアップでCOOを務めています。社員は8名、それに加えて**業務委託のエンジニア3名、デザイナー2名、マーケター1名**、さらに副業で手伝ってくれているメンバーも2名います。全員合わせると15人以上。少人数で最大の成果を出すことが求められる環境で、日々の業務はプロダクト開発の進行管理、営業チームとの連携、採用活動など多岐にわたります。

最近の仕事の状況は、とにかく「スピード」との戦い。Slackでのやり取りが中心ですが、「これお願い」「あれどうなった？」が飛び交い、**誰が何を持っているか把握しきれない**ことも。特に業務委託メンバーは週に数日しか稼働しないので、依頼の引き継ぎが曖昧になりやすいんです。

※参照ページ：https://www.tantan-task.com/（「チームステータス把握」機能）

#### 2. 直面している課題

実は、チーム内で「言った言わない」問題が頻発しています。

先日もこんなことがありました。業務委託のエンジニアに「先週お願いした機能改修、まだですか？」と聞いたら「え、聞いてないですけど…」と。Slackを遡ると確かに依頼はしていたのですが、**長いスレッドの中に埋もれていて**、相手には届いていなかったのです。

逆に、私が「やったって言いましたよね？」と言われたこともあります。チームの雰囲気も悪くなりかけていました。

スタートアップは信頼関係が命。でも今のやり方では、**関わる人が増えるほどコミュニケーションコストが増えていく**。特に業務委託やパートナーは「社外」の存在なので、社内ツールに招待するのも躊躇してしまう。もっと効率的にタスクを共有・管理する方法が必要でした。

※参照ページ：https://www.tantan-task.com/（「『言った言わない』をなくし、チームの生産性を最大化」）

#### 3. 解決への動機と行動

この問題を解決するため、タスク管理ツールを探し始めました。Asanaは機能が多すぎて、設定に時間がかかりそう。しかも業務委託メンバー全員にアカウント作ってもらうのは現実的じゃない。Trelloはシンプルだけど、相互にタスクを追加し合う機能がない。

あるとき、スタートアップ経営者の勉強会で「タンタン」の話を聞きました。「**URLを共有するだけで、業務委託メンバーもアカウント不要でタスク追加できる**」と。しかも「タスク内でチャットもできる」。まさに求めていた機能でした。

※参照ページ：https://www.tantan-task.com/（URL発行・共有機能、グループ連携機能）

#### 4. サービス検討のプロセス

重視したのは以下の点です：

1. **相互タスク追加**：メンバー同士でタスクを追加し合える
2. **業務委託も使いやすいか**：アカウント不要、URL共有だけで参加できる
3. **タスク内チャット**：タスクごとに会話が完結する
4. **進捗の可視化**：誰がどれくらいタスクを抱えているか把握できる

タンタンは全ての要件を満たしていました。業務委託やパートナーに「このURLから依頼追加してください」と伝えるだけで、**ツールの使い方を教える時間も不要**。Slack連携で通知もリアルタイムに受け取れる点も魅力でした。

※参照ページ：https://www.tantan-task.com/（「相互タスク追加」「タスク内チャット」「スマート通知」）

#### 5. 意思決定の理由

最終的な決め手は「**関わる人が多い組織でも、導入コストが低い**」ことでした。

社員、業務委託、パートナー、副業メンバー...雇用形態がバラバラでも、URLを共有するだけで全員が同じ土俵に乗れる。「言った言わない」がなくなれば、チームの雰囲気も良くなる。タスクの進捗が可視化されれば、負荷調整もしやすくなる。

導入後は、チーム全員でタンタンを使い、**誰が何を持っているか一目でわかる状態**を作りたいと考えています。

※参照ページ：https://www.tantan-task.com/

### このペルソナへのアプローチ指針
- **響くメッセージ**：「社員も業務委託も、URLを共有するだけでタスク追加」「言った言わない問題を解決」
- **重要なタイミング**：チーム拡大時、業務委託メンバー増加時、コミュニケーション問題発生時
- **提供すべき情報**：グループ連携機能の詳細、アカウント不要でのURL共有、Slack連携の設定方法

## ペルソナ3：複数クライアントを抱えるWeb系フリーランス

### 基本設定
- **業界**：Web制作、デザイン、エンジニアリング、ライティング、マーケティング支援
- **企業規模**：個人事業主（1人）
- **役職**：フリーランス（職種はさまざま）
  - Webデザイナー
  - フロントエンドエンジニア
  - Webライター / 編集者
  - マーケター / 広告運用者
  - Webディレクター
- **部門**：該当なし（個人事業）
- **関わる人**：5〜10社のクライアント、外注パートナー

### なぜこの人がターゲットか
- **複数クライアント = 依頼元が多様**
- クライアントごとにツールが違う（Slack、Chatwork、メール、電話...）
- 「依頼を受ける側」なので、自分から情報を拾いに行かないといけない
- 依頼の見落とし = 信頼損失 = 仕事がなくなる
- Web系だから、デジタルツールへの抵抗感が低い

### 購買ストーリー（約1,100文字）

#### 1. 日常の仕事と状況

私はフリーランスとして独立して3年目。**Web系の仕事**を中心に、現在常時5〜6社のクライアントと取引しています。日々の業務は制作・開発がメインですが、実際には見積もり作成、請求書発行、クライアントとのコミュニケーションなど、雑務も多いです。

最近の仕事の状況は、正直パンク寸前。**クライアントごとにコミュニケーション手段が違う**んです。A社はSlack、B社はChatwork、C社はメール、D社は電話で依頼してくる。それぞれのツールを開いて、依頼内容を確認して、自分のタスク管理ツールに転記して...。この作業だけで毎朝30分は消えています。

※参照ページ：https://www.tantan-task.com/（「チャット、メール、口頭での依頼。散らばったタスクを拾い集める時間」）

#### 2. 直面している課題

実は、先月「依頼の見落とし」で信頼を失いかけました。

あるクライアントから「先週お願いした修正、まだですか？」と連絡がありました。Chatworkを遡ると、確かに依頼がありました。でも**他のクライアントからの通知に紛れて見落としていた**のです。すぐに対応して事なきを得ましたが、冷や汗をかきました。

フリーランスにとって信頼は命。でも今のやり方では、いつまた同じミスをするかわからない。特に「期限」や「参考資料」の情報が依頼時に揃っていないことも多く、確認のやり取りだけで時間が取られます。

※参照ページ：https://www.tantan-task.com/（「細かい指示出しは不要」のメッセージ）

#### 3. 解決への動機と行動

この状況を変えるため、「依頼を一元化できるツール」を探し始めました。

最初はTrelloを使っていましたが、結局自分で登録する手間は変わらない。Notionも試しましたが、クライアントに使い方を説明するのが面倒でした。

あるとき、同業のフリーランス仲間から「タンタンいいよ」と聞きました。「URLを渡すだけで、クライアントが直接タスク追加してくれる」と。しかも「期限必須」などのルールを設定できるから、必要な情報が最初から揃うそうです。

※参照ページ：https://www.tantan-task.com/（URL発行・共有機能）

#### 4. サービス検討のプロセス

検討する中で、重視したのは以下の点です：

1. **クライアントに使ってもらえるか**：相手に負担をかけない簡単さ
2. **必要な情報が揃うか**：期限、参考URLなどを必須にできるか
3. **クライアント別に管理できるか**：プロジェクト別のURL発行

タンタンは「用途に合わせてURLを無制限に発行可能」とあり、**クライアント別・プロジェクト別に分けて管理できる**ことがわかりました。

※参照ページ：https://www.tantan-task.com/（「プロジェクト別、クライアント別など、用途に合わせてURLを無制限に発行」）

#### 5. 意思決定の理由

最終的な決め手は「シンプルさ」でした。

クライアントにURLを送るだけで、依頼フォームが開く。難しい説明は不要。**どんなツールを使っているクライアントでも、URLだけあれば依頼できる**。統一されたフォーマットで情報が整理されるから、「どこを見ればいい？」と迷う時間もなくなる。

導入後は、全クライアントに「ご依頼はこちらから」とURLを共有し、依頼の一元管理を実現したいです。

※参照ページ：https://www.tantan-task.com/（「シンプルだから、迷わない」）

### このペルソナへのアプローチ指針
- **響くメッセージ**：「URLを渡すだけで、クライアントからの依頼が自動で一覧に」
- **重要なタイミング**：依頼の見落としによるトラブル発生後、新規クライアント増加時
- **提供すべき情報**：クライアント別URL発行の使い方、ルール設定機能

## カスタマージャーニー作成のためのペルソナ情報

### 共通の購買プロセス
1. **日常業務の中での課題認識**：タスク管理の非効率さに気づく
2. **トリガーイベント**：タスク漏れによるトラブル、関わる人の増加
3. **情報収集開始**：「タスク管理ツール」で検索、知人への相談
4. **競合ツールの試用**：Asana、Trello、Backlog等を試す
5. **タンタンの発見**：口コミ、検索で発見
6. **機能比較・検討**：独自機能（URL共有タスク追加）の理解
7. **導入決定**：シンプルさと独自価値に納得
8. **運用開始**：URL発行・共有、関係者への展開

### 段階ごとのペルソナ差異
