# ポジショニング分析レポート：タンタン

## エグゼクティブサマリー

### 現在の市場における各社の位置づけ
- **Asana**: 大企業〜中堅企業向けの包括的ワークマネジメントプラットフォーム。PJ全体を俯瞰する管理者向け。
- **Backlog**: 開発チーム向けの統合プロジェクト管理ツール。Git連携やバグトラッキングなど技術者に特化。
- **Trello**: 個人〜チーム向けのシンプルなカンバンツール。視覚的で直感的、学習コストが低い。
- **Notion**: オールインワンワークスペース。自由度が高いが設計が必要。学習コスト高め。
- **Todoist**: 個人向けタスク管理の定番。シンプルだが「自分で管理」が前提で閉じている。
- **タンタン**: 「自分専用のタスク管理」なのに「設定なしで他者と繋がれる」唯一のツール。

### タンタンの差別化ポイント（3つの軸）

| 軸 | 内容 | 競合との違い |
|----|------|-------------|
| **自分が中心** | 自分専用のタスク管理ツール | 競合はPJ/チーム/ボードが中心 |
| **他者と繋がりやすい** | URLを共有するだけで他者がタスク追加可能 | 競合は招待・権限設定が必要 |
| **シンプル** | 複雑な設定不要、すぐに使い始められる | Asana/Notion等は学習コスト高い |

→ 「自分が中心」なのに「他者と繋がれる」という、一見矛盾する価値を両立しているのがタンタンの独自性

※参照情報源：https://www.tantan-task.com/, https://asana.com/ja, https://backlog.com/ja/, https://trello.com/ja, https://www.notion.so/ja, https://todoist.com/ja

## ポジショニング分析

### 軸1：視点の中心（自分が中心 vs プロジェクトが中心）

**競合ツール：プロジェクト/ボード/ワークスペースが中心**
```
┌─────────────────────┐
│   プロジェクト全体    │
│  ┌───┐ ┌───┐ ┌───┐  │
│  │タスク│ │タスク│ │自分│ ← 割り当てられる存在
│  └───┘ └───┘ └───┘  │
└─────────────────────┘
```
- Asana：プロジェクト単位で管理、タスクは割り当てられるもの
- Backlog：プロジェクトに参加してタスクを受け取る
- Trello：ボードが中心、カードを動かす
- Notion：ワークスペース/ページが中心

**タンタン：自分が中心**
```
      PJ-A    人-B
        ╲    ╱
        【自分】
        ╱    ╲
    人-C      PJ-D
```
- 自分が中心にいて、周りにいろんな人やPJがある
- プロジェクトやチームを設定しなくても他者と繋がれる

| ツール | 視点の中心 | 自分の立ち位置 |
|--------|----------|--------------|
| Asana | プロジェクト | 割り当てられる存在 |
| Backlog | プロジェクト | 課題を担当する存在 |
| Trello | ボード | カードを動かす存在 |
| Notion | ワークスペース | ページを編集する存在 |
| Todoist | 自分（閉じてる） | 自分で登録・管理 |
| **タンタン** | **自分（開いてる）** | **中心にいて、周りと繋がる** |

---

### 軸2：他者との繋がりやすさ

| ツール | 他者との繋がり方 | ハードル |
|--------|----------------|---------|
| Asana | チーム招待→アカウント作成→権限設定 | 高 |
| Backlog | プロジェクト招待→アカウント作成 | 高 |
| Trello | ボード共有→招待 | 中 |
| Notion | ページ共有→ゲスト招待 | 中 |
| Todoist | プロジェクト共有→招待 | 中 |
| **タンタン** | **URLを共有するだけ** | **低** |

→ タンタンは「設定なしで他者と繋がれる」唯一のツール

---

### 軸3：機能の複雑さ × 導入のしやすさ

```
        機能が複雑（学習コスト高）
                  ↑
         [Asana]    [Notion]
                  |
         [Backlog]
                  |
導入しやすい ←----|--------→ 導入に時間がかかる
                  |
   【タンタン】★  [Trello]  [Todoist]
                  |
                  ↓
        機能がシンプル（すぐ使える）
```

**分析**：
- **Asana/Notion**: 機能豊富だが学習コスト高い。設計・設定が必要
- **Backlog**: 開発チーム向け機能が多く、汎用的ではない
- **Trello/Todoist**: シンプルで使いやすい
- **タンタン**: 最もシンプルで導入しやすい。URLを発行するだけで即利用開始

---

### ポジショニングマップ（統合）

```
                他者と繋がりやすい
              （設定・招待不要）
                      ↑
                      |
                 【タンタン】★
                      |
自分中心 ←------------|---------------→ プロジェクト中心
（個人視点）          |              （管理者視点）
                      |
    [Todoist]         |     [Trello]      [Asana]
                      |     [Notion]    [Backlog]
                      |
                      ↓
              他者と繋がりにくい

```

**タンタンの独自ポジション**：
- 「自分中心」なのに「他者と繋がりやすい」
- これは他のツールにはない組み合わせ

## LP（ランディングページ）への示唆

### 訴求すべき差別化メッセージ

| 競合 | ユーザーの不満 | タンタンの訴求 |
|------|---------------|---------------|
| Asana | 機能が多すぎ、学習コスト高い | 「シンプルで迷わない」 |
| Backlog | 開発チーム向けで汎用的じゃない | 「誰でも使える」 |
| Trello | 結局自分で登録が必要 | 「みんなが追加してくれる」 |
| Notion | 設計が大変、自由度高すぎ | 「すぐ使える」 |
| Todoist | 他者からの依頼管理には向かない | 「依頼を受ける人のためのツール」 |

### 訴求内容

**訴求する**：
1. ✅「みんなが追加できる」（URLを共有するだけ）
2. ✅「タスク単位で会話できる」
3. ✅「シンプルさ」

**訴求しない**：
- ×「チームステータス」「進捗管理」（機能として存在しない）
- ×「プロジェクト管理」（Asana/Backlogとの差別化）

## カスタマージャーニー作成のためのポジショニング情報

### タンタンが選ばれるシナリオ

- **顧客タイプ**：人との関わりが多く、いろんな人/PJからの依頼が来る人
- **気づきのきっかけ**：タスク漏れによるトラブル、転記作業への疲弊
- **発想の転換ポイント**：「プロジェクト管理ツール」ではなく「自分中心のタスク管理」という選択

### 各段階での差別化ポイント

| 段階 | 競合との違い | 訴求ポイント |
|------|------------|-------------|
| 認知 | 具体的ペインポイント | 「タスクを拾い集める手間」 |
| 検討 | 独自機能 | 「URL共有だけでタスク追加」 |
| 決定 | 導入の容易さ | 「すぐ使える」「シンプル」 |

### 空白市場と競合領域

**タンタンの独自ポジション**：
- 「自分中心」なのに「他者と繋がりやすい」
- URLを共有するだけでタスク追加
- 「依頼を受ける側」視点のツール

**レッドオーシャン（競合激戦）**：
- 一般的な個人タスク管理（Todoist等）
- 大企業向けプロジェクト管理（Asana等）
- カンバン形式のタスク管理（Trello等）
