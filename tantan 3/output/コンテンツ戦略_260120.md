# tantan コンテンツ戦略

**作成日**: 2026年1月20日
**背景**: クライアントとの会話で決まった方向性を整理

---

## エグゼクティブサマリー

### 決定事項
1. **機能ページを分割・詳細化する**（SEO + LLMO対策）
2. **深掘り記事を作成する**（ユースケース・解決策の提示）
3. **SEOを強化する**（ドメインパワーの底上げ）
4. **カスタマージャーニー上のコンテンツを網羅的に作成する**

### 優先順位
| 優先度 | 施策 | 理由 |
|--------|------|------|
| 高 | 機能ページの分割 | SEO/LLMO両対策の基盤 |
| 高 | 深掘り記事の作成 | ターゲットの文脈に刺さる |
| 中 | SEO強化（ビッグワード対策） | ドメインパワー向上 |
| 低〜中 | FAQ | やらないと決める理由はないが最優先ではない |

## 1. 現状の課題

### 機能ページの問題
- 機能が一覧で羅列されているだけ
- LLMや検索エンジンが「その機能はない」と誤認するリスク
- **事例**: スマート書記では一覧にある機能が「ない」と判断された

### LLMO対策の前提
- **AIは検索結果の上位から情報を参照する**
- つまりSEOが弱いと、AIにも引用されない
- まずSEOで評価される構成にすることが必要

---

## 2. 機能ページの分割・詳細化

### 方針
**1機能1ページ**（または詳細なFAQページ）として切り出す

### 作成すべき機能紹介ページ

| 機能名 | 概要 | ターゲットの課題 |
|--------|------|-----------------|
| **依頼URL機能** | アカウントがない外部の人（クライアント等）にもタスクを追加してもらえる | クライアントからの依頼が散らばる |
| **タスク内チャット機能** | タスクに関するやり取りをタスク内で完結（Slackで探す時間をゼロに） | 12秒 vs 30秒の差、情報が埋もれる |
| **グループとルール設定** | プロジェクトごとにルール（納品場所など）を固定 | 毎回同じ説明をするコスト |
| **タスク完了確認・アラート** | 未対応者への催促を自動化 | 集計・連絡で1週間かかる業務 |
| **チームステータス把握** | 誰がどれくらいタスクを抱えているか可視化 | 負荷の偏り、「言った言わない」 |

### ページ構成案（各機能ページ）

```
1. 機能の概要（一言で何ができるか）
2. こんな課題を解決します（ペルソナの悩み）
3. 具体的な使い方（スクリーンショット付き）
4. 導入事例 / ユースケース
5. 関連機能へのリンク
6. CTA（無料で試す）
```

## 3. 深掘り記事の作成

### 方針
単なる機能説明ではなく、**「誰のどんな痛みを解決するか」**というストーリーを持たせる

### 作成すべき記事

#### バックオフィス向け
| 記事タイトル案 | 切り口 | 紹介する機能 |
|---------------|--------|-------------|
| 未対応者への催促コストを削減する方法 | 集計・連絡で1週間かかる業務を自動化 | タスク完了確認・アラート |
| PCセキュリティチェックの回収を効率化 | 定期タスクの回収自動化 | ルール設定、アラート |

#### PM / 支援会社向け
| 記事タイトル案 | 切り口 | 紹介する機能 |
|---------------|--------|-------------|
| ツールが分散してタスクが見えなくなる問題を解決 | Slack、Chatwork、Backlog…の一元化 | 依頼URL、グループ管理 |
| クライアントワークの「タスク転記」をなくす方法 | 週5時間の名もなき作業を削減 | 依頼URL |
| 「言った言わない」をなくすタスク管理 | チームの信頼関係を守る | タスク内チャット |

#### 検証・やってみた系
| 記事タイトル案 | 切り口 |
|---------------|--------|
| tantanの依頼URL機能を使ってみた | 実際の業務でどう変化したか |
| 〇〇（機能名）を1週間試してわかったこと | 導入のリアルな体験談 |

## 4. SEO強化

### 方針
- LLMに参照されるためには、まず**検索エンジンで上位表示される**ことが前提
- ターゲットが直接検索しない一般ワードでも、サイト評価を上げる手段として作成

### 対策すべきキーワードカテゴリ

| カテゴリ | 例 | 目的 |
|---------|-----|------|
| ビッグワード | タスク管理、業務効率化 | ドメインパワー向上 |
| トレンドワード | AI×タスク管理 | 話題性でトラフィック獲得 |
| 課題系ワード | タスク漏れ 防止、言った言わない 解決 | ターゲットの課題認識段階を獲得 |

### 既存のKW戦略との連携
`KW戦略_選定サマリー.md` で選定済みの90キーワード（課題認識30件、情報収集30件、比較検討30件）を活用

## 5. カスタマージャーニー上のコンテンツ

### 方針
「ターゲットは検索しないのでは？」という懸念があっても、**やらないと決める理由はない**

### 理由
- ユーザーは最終的な答えにたどり着く過程で様々なリサーチを行う
- その「文脈（コンテキスト）」に接点を持つために、FAQレベルでも広めにコンテンツを出す
- AI時代のリサーチ対策としても有効

### tantanのターゲット文脈

| ターゲット | 検索/質問する文脈 | コンテンツ例 |
|-----------|------------------|-------------|
| PM / 支援会社 | 商談を効率化するには？ | 「商談後のタスク整理を自動化」記事 |
| PM / 支援会社 | タスクの抜け漏れを防ぐには？ | FAQ: 抜け漏れ防止のベストプラクティス |
| バックオフィス | 催促業務を減らすには？ | 「未対応者への催促自動化」記事 |
| フリーランス | 複数クライアントの管理方法 | 「クライアント別URL発行」の使い方 |
